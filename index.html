Insider Trades and Stock Performance¶

There are many reasons why insiders trade their company stock and It's not correct to assume it's because of a particularly bad/good forecast for a company's earnings or some other reason tied to the underlying value of their company. However, there might be some loose association for trades of particular kinds. For example, around half the trades that are reported from Google are a result of direct sales of stock, not option exercise which is often necessary to do because those expire anyway. Sales that aren't option exercises aren't necessary so they might be for any number of reasons, one could be a belief that the value of keeping shares has in some way diminished.

In this project, I set out to accomplish a few goals. The first is to extract insider trading data for a company, Google as a first example, fit the data into a dataframe and explore the relationship between the insider trade type, trader, trade date and the actual performance of the stock itself. I can then compare the performance of traders at different companies.

I will mainly focus on stock sales and not option exercises or stock purchases to simplify the analysis and because option exercises are mostly tied to a date and not indicative of intent and share purchases are rarely reported in my data.

Data Extraction

There are many ways to get insider trading data, either directly from the SEC website or through EDGAR API. One shortcoming of the SEC website is that it requires the CIK identifier for a public company to access the website its trades are reported in. This isn't too much of a hassle, since the CIK is publicly known, but it is annoying to deal with. The bigger issue it there is often a reliability problem when doing an automated scraping of the website since the sec, for good reason doesn't want to overwhelm their servers with requests from bots.

Another option is the Edgar API which is provided by the SEC and has a good request limit of ~10 a second. But the data is returned as JSON which can be decoded into a dataframe, however, there were some issues that I had with this method. Certain columns would be missing or data wasn't read into frames the way I had thought it should.

The final option is to scrape from a third-party site. I chose open insider because it made querying for range effortless and had all the data on trades that I wanted in a table format.

Data Parsing

I had an issue with reading the web page directly into a table using pandas because the read-in frame had wrong column names and some data was not read from the table at all. Instead, I decided to find the table rows and build an array for that row using the td tags as values. Then I added this array to a 'cells' array which was my dataframe in a 2d array format. Pandas was able to decode this into a dataframe very easily.

In [157]:
"""
This method will take a response object 
and extract the contents of the first 
table it finds into a dataframe and then return that 
dataframe.
"""
def get_cells(page):
  soup = BeautifulSoup(page.text)
  table = soup.find('table', {'class':'tinytable'})
  header = table.find('thead')
  headers = [th.text.replace(u'\xa0', u'') for th in header.find_all('th')]
  rows = table.find_all('tr')
  cells = []

  for row  in rows:
    lst = row.find_all('td')
    if lst :
      cells.append([td.text for td in lst])
  return (headers, cells)

Data Management

After I had the data available in a dataframe I had to manage the data so that it was in the right format. For quantity sold and price, I had to remove the commas to convert the columns to numeric values. I also noticed that quantity was negative if the trade was a sale of shares so I created a new column with the absolute value of the quantity sold and another column using the proportion of the quantity sold for each trade relative to all the quantities sold. The second new column makes it easier to plot the quantity data. I also added a color column for when it comes to plot the trade as a sale/option exercise/buy.

def format_df(df):
  df['Qty'] = pd.to_numeric(df['Qty'].str.strip().str.replace(",", ""))
  df['TradeDate'] =pd.to_datetime(df['TradeDate'])
  df['Color'] = df.apply(get_color , axis=1)
  df['AbsQty'] = df['Qty'].abs()
  totalQty = df['AbsQty'].sum()
  df['AbsQty8k'] = df.apply(lambda x : (x['AbsQty']/totalQty)*8000, axis=1)
  df['Price'] = df.apply(lambda x: Decimal(sub(r'[^\d.]', '', x['Price'])), axis=1)
  return df

"""
This method takes the splits a 
stock has done and applies them to
legacy stock values that need to
be updated
"""
def adj_split(x, splits):
  split_adj = 1
  trade_date = x["TradeDate"]
  for idx in splits.index:
    if(trade_date < idx.tz_localize(None)):
      split_adj *= splits[idx] 
  return split_adj

Exploratory Data Analysis

We can more formally test to see if the trades benefited the insider by looking at the general trend of the stock around 4 months after a sale of stock. The way I did it was by taking the date of trade finding the line of best fit for the next 4 months of stock data and seeing if the coefficient which is essentially the trend was negative in which case it was a good idea to sell, or if it was positive in which case it was a bad time to sell.

"""
This method returns the coefficient of the 
regression line between x and y
"""
def lin_reg(x, y):
  x =	np.array(x).reshape(-1, 1)
  y =	np.array(y).reshape(-1, 1)
  fit = LinearRegression().fit(x, y)
  return fit.coef_[0][0]

"""
This method arranges dates and opening
stock values to get their regression coefficient
"""
def get_st_perf(x):
  p = x['AdjPrice']
  d = x['TradeDate']
  s = d - relativedelta(days=+1)
  e = d + relativedelta(months=+4)
  valid_dates = list(filter(lambda x : s <= x and x <= e, df_stock.index))
  dates_in_range = df_stock.loc[valid_dates]
  coef = lin_reg([i for i in range(len(valid_dates))], list(dates_in_range['Open']))
  return coef
def get_dist_maxp(x):
  coef = get_st_perf(x)
  return coef

Hypothesis Testing

We can test the relationship between share sales and the coefficient of the trend line for the stock in the months after the trade using the quantity of stock sold. We can test to see if the indirect measure of confidence in the sale through quantity sold translates to a loose prediction of the performance.

The results show a negative correlation between the number of shares sold and the performance of the stock. However, the p-value for the correlation is very high so I have failed to reject the null hypothesis that the quantity of sold shares has an impact on the future performance of the stock.

Conclusion

Looking at the performance of stocks and the insider trades, insider trades don't seem to be a very good perdictor of wheather an insider trade will really tell you anything about the oncoming performance of the stock. For Google stock it appears to be fairly decent in terms of the perdicitve power but Microsoft is more volitile. This makes sense since insider traders could be trading for a number of reasons (financing charitable organizations, buying a home, emergency reserves, etc). Even the quantity traded might not be enough to indicate any fears an insider may have about their company's future as shown through OLS. There might be other ways to assess the perdictive power of insiders but underlying company performance will likely trump a murky trade.
